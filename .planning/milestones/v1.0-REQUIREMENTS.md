## v1 Requirements (Complete)

### Authentication

- [x] **AUTH-01**: User can sign in with Google
- [x] **AUTH-02**: User can sign in with Facebook
- [x] **AUTH-03**: User stays logged in across browser refresh
- [x] **AUTH-04**: User can sign out
- [x] **AUTH-05**: Unauthenticated users redirected to login for protected routes

### Data Isolation

- [x] **DATA-01**: User sees only their own grocery lists
- [x] **DATA-02**: User's language preference is per-user
- [x] **DATA-03**: User's theme preference is per-user
- [x] **DATA-04**: User's address is per-user
- [x] **DATA-05**: System settings (scraper config) unaffected by auth
- [x] **DATA-06**: Existing grocery lists are claimable after migration

### Sync

- [x] **SYNC-01**: List changes sync to other devices within 5 seconds
- [x] **SYNC-02**: Tabs on same device sync near-instantly
- [x] **SYNC-03**: Simultaneous edits show conflict notification
- [x] **SYNC-04**: Offline: last-cached list shown read-only
- [x] **SYNC-05**: Polling doesn't degrade SQLite performance

### Sessions

- [x] **SESS-01**: User sees active sessions with device names
- [x] **SESS-02**: User can sign out individual sessions
- [x] **SESS-03**: Same email via Google + Facebook links to one account
- [x] **SESS-04**: Auth errors show user-friendly messages
- [x] **SESS-05**: New users see appropriate empty states

(None defined)

## Out of Scope

| Feature | Reason |
|---------|--------|
| Email/password login | OAuth-only for maximum seamlessness |
| WebSocket real-time sync | Polling + SSE sufficient for grocery list use case |
| Offline write capability | Complexity not justified; read-only cache acceptable |
| User dietary preferences | Already per-list, not needed at user level |
| Mobile app | Web-first |
