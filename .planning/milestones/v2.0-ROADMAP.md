# Milestone v2.0: Krepza Rebrand + Ad Monetization

**Status:** ✅ SHIPPED 2026-04-29
**Phases:** 5-7
**Total Plans:** 6

## Overview

Full rebrand from "LT Grocery" to "Krepza" with Google AdSense monetization and domain-agnostic deployment. Replaced all branding (i18n, metadata, logos, static pages), wired Google AdSense into 5 ad component types with GDPR consent banner, and configured deployment to work on any domain via env vars.

## Phases

### Phase 5: Complete Rebrand

**Goal**: Replace every "LT Grocery" reference with "Krepza" — i18n, metadata, layout, static pages, favicons, logo.
**Depends on**: None
**Plans**: 3 plans

Plans:
- [x] 5-1: i18n + Metadata Rebrand (BRAND-01,02,03)
- [x] 5-2: Static Pages + Scraper User-Agent (BRAND-04,05)
- [x] 5-3: Logo + Favicons (BRAND-06,07)

**Success criteria:**
1. No string "LT Grocery" appears anywhere in the running app UI
2. Browser tab shows "Krepza - Price Checker" favicon
3. Desktop sidebar displays Krepza logo SVG instead of text/emoji
4. Privacy policy and data deletion pages reference "Krepza" not "LT Grocery"
5. Scraper identifies as "krepza/2.0" in User-Agent

### Phase 6: AdSense Integration

**Goal**: Google AdSense ads render on all pages via the 5 existing ad component types, with GDPR-compliant consent and publisher-ID-driven conditional loading.
**Depends on**: Phase 5
**Plans**: 2 plans

Plans:
- [x] 6-1: AdSense Script + GDPR Consent + Env Var (ADS-06,07,08)
- [x] 6-2: Wire All 5 Ad Components (ADS-01,02,03,04,05)

**Success criteria:**
1. Ads render in AdBanner, AdLeaderboard, AdNativeCard, AdSideRail, AdSponsoredRow components
2. Setting `NEXT_PUBLIC_ADSENSE_ID` to a valid publisher ID causes real ads to load
3. Omitting `NEXT_PUBLIC_ADSENSE_ID` causes all ad slots to render nothing (no fallback)
4. GDPR consent banner appears before any ad script loads
5. Existing adblock detection and dismiss behavior preserved in AdSlot.tsx

### Phase 7: Domain-Agnostic Deployment

**Goal**: App deploys on any domain by changing only env vars in docker-compose.yml — no hardcoded URLs remain.
**Depends on**: Phase 5, Phase 6
**Plans**: 1 plan

Plans:
- [x] 7-1: Domain-Agnostic Deployment Config (DEPLOY-01,02,03)

**Success criteria:**
1. Changing `NEXTAUTH_URL` env var in docker-compose.yml changes all dynamic URL references
2. `docker compose up` builds and serves Krepza with all branding and ads
3. `.env.example` documents `PUBLIC_URL`, `NEXT_PUBLIC_ADSENSE_ID`, and all required vars

## Milestone Summary

**Key Decisions:**
- D-01: Inline SVG component for Krepza logo (consistent with lucide-react pattern)
- D-02: Full legal page rewrite for privacy/TOS (not just text replacement)
- D-03: Scraper git bot identity stays as-is (infrastructure, not branding)
- D-04: AdSense via `next/script` with `afterInteractive` strategy
- D-05: Lightweight custom GDPR consent banner (no external library)
- D-06: NEXTAUTH_URL as canonical domain config (PUBLIC_URL as documented alias)

**Issues Resolved:**
- Hardcoded "LT Grocery" in client-providers.tsx (now uses useI18n())
- Desperate begging copy in ad slots (replaced with neutral adblock notice)
- No AdSense in docker-compose.yml (added NEXT_PUBLIC_ADSENSE_ID)

**Issues Deferred:**
- AdSense Auto Ads (auto-inserted page ads) — manual slots preferred
- Multiple ad networks — AdSense covers 90%+ market
- Ad performance dashboard — analytics for future milestone
- PWA manifest / service worker — web-first deployment

**Technical Debt Incurred:**
- `pleaseWhitelist` and `pleaseWhitelistDesperate` i18n keys preserved but repurposed — could be cleaned up
- Consent banner uses page reload for activation — smoother approach (dynamic script injection) deferred
