<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Internet → Cloudflare Edge | Untrusted public traffic enters Cloudflare's edge network. Cloudflare provides DDoS, WAF, SSL termination. |
| Cloudflare Edge → cloudflared container | Encrypted outbound-initiated QUIC/HTTP2 tunnel. Authenticated via TUNNEL_TOKEN. |
| cloudflared → web:3131 | Internal Docker network traffic (unencrypted HTTP). Both containers on same bridge network. |
| .env file → filesystem | Secrets at rest on VPS. Protected by filesystem permissions and gitignore. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-01 | Information Disclosure | TUNNEL_TOKEN in .env | mitigate | .env is gitignored (confirmed in .gitignore line 7). .env.example uses empty placeholder with WARNING comment. Real token lives only in .env on VPS. If leaked: rotate token in Cloudflare dashboard (Zero Trust → Networks → Tunnels → Configure → Regenerate). |
| T-08-02 | Spoofing | Unauthorized tunnel connection via leaked TUNNEL_TOKEN | mitigate | Same as T-08-01. Token rotation renders leaked token useless. Cloudflare audit log tracks tunnel connection IPs. |
| T-08-03 | Information Disclosure | NEXTAUTH_URL=http:// in production | mitigate | NEXTAUTH_URL set to https://krepza.lt in .env.example. OAuth providers reject http callbacks when https is configured. Session cookies require Secure flag which browsers only set over https. |
| T-08-04 | Denial of Service | cloudflared unreachable | accept | cloudflared connects outbound to Cloudflare — no inbound port needed. Docker auto-restart via `restart: unless-stopped`. Cloudflare edge absorbs DDoS. Single point of failure acceptable for single-server self-hosted app. |
| T-08-05 | Tampering | MITM between cloudflared and web container | accept | Internal Docker bridge network. Attacker would need host-level access (root compromise). Docker network isolation is sufficient for single-server threat model. |
</threat_model>
