Create two static files in packages/web/public/:

**1. public/ads.txt** — AdSense publisher verification file.
Per AdSense requirements (FEATURES.md § Table Stakes), the file format is:
```
google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0
```
- Use the literal template value `pub-XXXXXXXXXXXXXXXX` — this will be replaced by the actual publisher ID at deployment time via a sed or envsubst step, OR the user can replace it manually before deploy.
- The literal `f08c47fec0942fa0` is the standard AdSense certification authority ID and never changes.
- One line only, no trailing blank line.
- Next.js automatically serves files from `public/` with correct Content-Type (text/plain for .txt files) per D-01 discretionary note.

**2. public/robots.txt** — Search engine crawl rules.
Per FEATURES.md Differentiators § robots.txt:
```
User-agent: *
Allow: /$
Allow: /ads.txt
Disallow: /api/
Disallow: /_next/
Disallow: /auth/
```
- `Allow: /$` permits homepage crawling (exact root path).
- `Allow: /ads.txt` is CRITICAL per PITFALLS.md Pitfall #11 — Google's AdSense crawler uses Googlebot user-agent. If /ads.txt were disallowed, site verification would fail.
- `Disallow: /api/` blocks API route crawling (waste of crawl budget, may expose dynamic content).
- `Disallow: /_next/` blocks Next.js internal assets (JS chunks, etc. — not useful for search indexing).
- `Disallow: /auth/` blocks auth-related paths.
  </action>

  <action>
Add the AdSense site verification meta tag to the existing `metadata` export in `packages/web/src/app/layout.tsx`.

Per CONTEXT.md Agent's Discretion: "AdSense site verification via meta tag in layout.tsx (cleaner than HTML file approach)".

The AdSense verification meta tag format is:
```html
<meta name="google-adsense-account" content="ca-pub-XXXXXXXXXXXXXXXX">
```

In Next.js App Router Metadata API, add this to the `metadata` export using the `other` field:

```typescript
export const metadata: Metadata = {
  title: "Krepza - Price Checker",
  description: "Krepza — Lithuanian grocery price comparison and shopping list tool",
  icons: {
    icon: [
      { url: "/favicon.ico", sizes: "any" },
      { url: "/icon.svg", type: "image/svg+xml" },
    ],
    apple: { url: "/icon.png" },
  },
  // AdSense site verification — only included when publisher ID is configured
  // The `other` field renders arbitrary <meta> tags in <head>
  other: process.env.NEXT_PUBLIC_ADSENSE_ID
    ? { "google-adsense-account": `ca-${process.env.NEXT_PUBLIC_ADSENSE_ID}` }
    : undefined,
};
```

**Key implementation details:**
- Use `NEXT_PUBLIC_ADSENSE_ID` env var (same var already consumed by the existing `<Script>` for adsbygoogle.js on line 29 of layout.tsx).
- The meta tag `name` is `"google-adsense-account"` and `content` is `"ca-pub-XXXXXXXX"` (the `ca-` prefix + publisher ID).
- Conditionally include via ternary: when env var is set, include the tag; when empty (development, or before publisher ID configured), omit it entirely. This prevents an empty/broken verification tag.
- The `other` field in Next.js Metadata API accepts `Record<string, string>` and renders each key-value pair as `<meta name="{key}" content="{value}" />`.
- Do NOT modify the existing Script tag, icons configuration, or title/description.
  </action>

| T-09-01 | Information Disclosure | public/ads.txt | accept | ads.txt is DESIGNED to be public — it contains only the AdSense publisher ID which is already public information. This is the intended behavior per Google AdSense specification. |
| T-09-02 | Information Disclosure | public/robots.txt | accept | robots.txt is intentionally public for crawler consumption. API route paths are already discoverable through app navigation. No sensitive paths exposed. |
| T-09-03 | Information Disclosure | layout.tsx metadata | mitigate | The `other` metadata field conditionally renders `google-adsense-account` only when `NEXT_PUBLIC_ADSENSE_ID` is set.
