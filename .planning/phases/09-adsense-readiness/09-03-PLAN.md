  <how-to-verify>
**Prerequisites (before starting):**
- Phase 8 must be complete — `https://krepza.lt` must be accessible over HTTPS via Cloudflare Tunnel
- Your real AdSense publisher ID (format: `pub-XXXXXXXXXXXXXXXX`) — find it at https://adsense.google.com → Account → Settings → Account information
- Access to the deployment server to edit `.env`

---

**Step 1: Set the real publisher ID in .env**

On the deployment server, edit the `.env` file and set:
```
NEXT_PUBLIC_ADSENSE_ID=pub-XXXXXXXXXXXXXXXX
```
Replace `pub-XXXXXXXXXXXXXXXX` with your actual AdSense publisher ID from the dashboard.

**Step 2: Rebuild and redeploy**

After setting the env var, rebuild the Docker image and redeploy:
```bash
docker compose up -d --build web
```

**Step 3: Verify ads.txt is accessible**

After redeploy, verify from any machine:
```bash
curl -s https://krepza.lt/ads.txt
```
Expected output: `google.com, pub-XXXXXXXXXXXXXXXX, DIRECT, f08c47fec0942fa0`
(Should now show YOUR real publisher ID, not the template value)

**Step 4: Verify CSP and security headers in production**
```bash
curl -sI https://krepza.lt | grep -iE "content-security-policy|x-frame|x-content-type|referrer-policy|permissions-policy|strict-transport"
```
Expected: 6 headers present, including `Content-Security-Policy` (NOT `Report-Only`).

**Step 5: Verify site ownership meta tag in page source**

Visit `https://krepza.lt` in a browser, right-click → View Page Source, and search for:
```html
<meta name="google-adsense-account" content="ca-pub-XXXXXXXXXXXXXXXX">
```
This tag must be present in the `<head>` with your publisher ID.

**Step 6: Check for CSP violations in DevTools**

Open DevTools Console (F12 → Console) on `https://krepza.lt`. Look for red CSP violation messages mentioning:
- `pagead2.googlesyndication.com`
- `googleads.g.doubleclick.net`
- `tpc.googlesyndication.com`

There should be ZERO CSP violations for these domains. If violations appear, the CSP configuration in Plan 09-02 may need adjustment — report the exact violation messages.

**Step 7: Submit site to AdSense**

1. Go to https://adsense.google.com → Sites → Add site
2. Enter `krepza.lt` as the site URL
3. Click "Save and continue"
4. AdSense will verify site ownership via:
   - The `ads.txt` file at `https://krepza.lt/ads.txt` (created in Step 3 above)
   - The site verification meta tag (created by Plan 09-01)
5. Wait for the verification check to complete (usually a few minutes)
6. Once verified, the site status changes to "In review"

**Step 8: Confirm "In review" status**

In AdSense dashboard → Sites → krepza.lt, verify the status shows:
- ✅ Site ownership: Verified
- 📋 Review status: In review

---

**⚠ Pitfall #7 Warning:** AdSense may reject the site if the homepage shows no meaningful content to logged-out visitors. Before submitting, verify in an incognito window that `https://krepza.lt` shows product listings or a clear value proposition — not just a login wall. If the homepage is content-poor when not logged in, consider adding a public landing page before submitting.

**⚠ Pitfall #9 Warning:** If `NEXT_PUBLIC_ADSENSE_ID` is empty or not set, the existing conditional logic in `AdSlot.tsx` returns `null` for all ad components — zero ads will render. Double-check Step 1 and Step 3.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| .env → Docker build | NEXT_PUBLIC_ADSENSE_ID is compiled into the Next.js client bundle at build time. Must not be committed to git. |
| User → AdSense dashboard | Human must authenticate to Google AdSense and submit the site. No automated API exists for this. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-10 | Information Disclosure | .env file | mitigate | `.env` is gitignored already (confirmed in Phase 8). The publisher ID is not a secret (it appears publicly in ads.txt and ad units), but the .env file may contain other secrets (AUTH_SECRET, OAuth credentials). The agent does NOT read or modify .env — only guides the user through editing it. |
| T-09-11 | Denial of Service | AdSense submission | accept | AdSense site review is a manual Google process. Review can take days to weeks (Pitfall #7, BLOCKER in STATE.md). This is outside our control. The site is submitted early (Phase 9) so review runs in parallel with Phase 10-11. |
| T-09-12 | Repudiation | Publisher ID mismatch | mitigate | Task 1 verification ensures ads.txt template format is correct before deployment. The checkpoint ask confirms the user replaces the template `pub-XXXXXXXXXXXXXXXX` with the real ID. If the template value reaches production, ad requests will fail with invalid publisher ID errors visible in Network tab. |
</threat_model>
