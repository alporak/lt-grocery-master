**Implementation:**

Replace the existing `module.exports = nextConfig;` with an extended config that includes the headers function. The CSP string must be a single semicolon-separated string.

```javascript
/** @type {import('next').NextConfig} */
const nextConfig = {
  output: "standalone",
  images: {
    remotePatterns: [
      { protocol: "https", hostname: "firebasestorage.googleapis.com" },
      { protocol: "https", hostname: "storage.googleapis.com" },
      { protocol: "https", hostname: "cdn.barbora.lt" },
      { protocol: "https", hostname: "rimibaltic-res.cloudinary.com" },
    ],
  },
  async headers() {
    return [
      {
        source: "/:path*",
        headers: [
          {
            key: "Content-Security-Policy",
            value: [
              "default-src 'self'",
              "script-src 'self' 'unsafe-inline' 'unsafe-eval' https://pagead2.googlesyndication.com https://www.googletagmanager.com",
              "frame-src https://googleads.g.doubleclick.net https://tpc.googlesyndication.com",
              "connect-src 'self' https://pagead2.googlesyndication.com https://www.google.com",
              "img-src 'self' data: https:",
              "style-src 'self' 'unsafe-inline'",
              "font-src 'self'",
              "object-src 'none'",
              "base-uri 'self'",
              "form-action 'self'",
              "frame-ancestors 'self'",
            ].join("; "),
          },
        ],
      },
    ];
  },
};

module.exports = nextConfig;
```

**Why each directive:**

| Directive | Value | Reason |
|-----------|-------|--------|
| `default-src 'self'` | Only allow same-origin by default | Security baseline — anything not explicitly allowed is blocked |
| `script-src` | `'self'` + `'unsafe-inline'` + `'unsafe-eval'` + AdSense + GTM | Next.js needs inline/eval for hydration. AdSense script loads from pagead2.googlesyndication.com. GTM for potential future analytics. |
| `frame-src` | googleads.g.doubleclick.net + tpc.googlesyndication.com | These are the two origins AdSense uses for ad iframes (D-05). Without these, ad iframes are blocked. |
| `connect-src` | pagead2.googlesyndication.com + google.com | AdSense makes XHR/fetch connections to these origins for ad serving and reporting (D-05). |
| `img-src` | `'self'` + `data:` + `https:` | `https:` covers all HTTPS CDN image sources (grocery product images from barbora.lt, cloudinary.com, etc.). `data:` for inline favicon/base64. |
| `style-src` | `'self'` + `'unsafe-inline'` | Next.js CSS-in-JS requires inline styles |
| `font-src 'self'` | Self-hosted fonts only | Next.js App Router with next/font/google self-hosts at build time |
| `object-src 'none'` | Block all plugins | Flash/Java/ActiveX are obsolete attack vectors |
| `base-uri 'self'` | Prevent base tag injection | Attackers cannot redirect relative URLs |
| `form-action 'self'` | Prevent form hijacking | Forms can only submit to same origin |
| `frame-ancestors 'self'` | Allow same-origin framing only | Modern equivalent of X-Frame-Options. Allows ad iframes hosted on the same page while preventing external sites from framing krepza.lt |

**Critical decisions:**
- CSP is **enforcing** (not report-only) per D-02 — "CSP enforced immediately (not report-only)"
- `'unsafe-inline'` is REQUIRED for Next.js to function — removing it breaks the app itself
- `'unsafe-eval'` is included for Next.js runtime compatibility
- `img-src https:` uses the broad `https:` scheme source (not individual CDN hostnames) because grocery product images come from various CDN origins configured in `images.remotePatterns`. This is intentional: the security value of CSP is in blocking script injection, not enumerating every image CDN.
- The `source: "/:path*"` pattern applies to ALL routes — CSP must be consistent site-wide.

**Header justifications:**

| Header | Value | Reason |
|--------|-------|--------|
| `X-Frame-Options` | `SAMEORIGIN` | **D-03 explicit decision.** Allows page to host its own iframes (AdSense ads) while preventing external sites from framing krepza.lt. `DENY` would block AdSense iframes (Pitfall #2). `frame-ancestors 'self'` in CSP provides the modern equivalent. |
| `X-Content-Type-Options` | `nosniff` | Prevents MIME-type sniffing attacks. Required by security scanners and best practices. |
| `Referrer-Policy` | `strict-origin-when-cross-origin` | Sends full URL as referrer for same-origin requests, origin-only for cross-origin HTTPS requests, nothing for HTTP. Balances privacy with analytics utility. |
| `Permissions-Policy` | `camera=(), microphone=(), geolocation=()` | Explicitly denies browser API access for camera, microphone, and geolocation. Krepza does not use any of these. Denying them prevents potential abuse. |
| `Strict-Transport-Security` | `max-age=63072000; includeSubDomains; preload` | HSTS tells browsers to always use HTTPS for this domain for 2 years (63072000 seconds). `includeSubDomains` covers all subdomains. `preload` allows submission to browser HSTS preload lists. Cloudflare already provides HTTPS, but HSTS at the application layer is defense-in-depth. |

**X-Frame-Options + frame-ancestors coexistence:**
- Both `X-Frame-Options: SAMEORIGIN` and `frame-ancestors 'self'` (in CSP from Task 1) are present.
- Modern browsers respect `frame-ancestors` in CSP, older browsers fall back to `X-Frame-Options`.
- Both set to the same effective policy (same-origin-only framing).
- This is intentional redundancy, not a conflict.

**Do NOT add these headers (anti-patterns to avoid):**
- `X-XSS-Protection: 1; mode=block` — deprecated, can introduce XSS vulnerabilities in some browsers. CSP replaces it.
- `X-Download-Options: noopen` — IE8-only, irrelevant for modern browsers.
- `X-Powered-By: ...` — should be REMOVED if present, not added. Next.js may add this by default; if it appears in responses, suppress it but don't add it here.
  </action>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Browser → Next.js | HTTP responses carry CSP + security headers that browsers enforce. This is where the security policy takes effect. |
| Next.js → CDN origins | Product images loaded via `img-src https:` allow any HTTPS source. Accepting this as a trade-off for grocery product image flexibility. |
| Next.js → AdSense | Scripts, frames, and connections to AdSense domains are explicitly allowed in CSP. These are the monetization trust boundary. |
| Client → krepza.lt | `frame-ancestors 'self'` + `X-Frame-Options: SAMEORIGIN` prevent external sites from framing krepza.lt in clickjacking attacks. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-09-04 | Spoofing | CSP header | mitigate | CSP `default-src 'self'` prevents loading scripts/styles/images from untrusted origins. Script injection via XSS is blocked because inline scripts from unknown sources are denied (except the explicitly allowed AdSense and GTM origins). `'unsafe-inline'` is required for Next.js runtime but the attack surface is limited because Next.js controls what inline scripts it emits. |
| T-09-05 | Tampering | CSP header | mitigate | `frame-ancestors 'self'` prevents clickjacking where an attacker frames krepza.lt on a malicious site to trick users. `form-action 'self'` prevents form data exfiltration via form hijacking. `base-uri 'self'` prevents base tag injection that could redirect relative URLs to attacker-controlled domains. |
| T-09-06 | Information Disclosure | Referrer-Policy, HSTS | mitigate | `Referrer-Policy: strict-origin-when-cross-origin` limits referrer information sent to third parties (origin only for cross-origin). `Strict-Transport-Security` with `max-age=63072000` and `preload` ensures all future visits use HTTPS — never downgrade to HTTP. |
| T-09-07 | Denial of Service | CSP misconfiguration | mitigate | CSP is tested to include ALL required AdSense origins (Pitfall #1 prevention). `'unsafe-inline'` is present for Next.js runtime. Without these, the app would break (scripts blocked) or ads would not load (revenue blocked). The `source: "/:path*"` pattern applies consistently to all routes — no route is accidentally unprotected or over-protected. |
| T-09-08 | Information Disclosure | Permissions-Policy | mitigate | `Permissions-Policy: camera=(), microphone=(), geolocation=()` explicitly denies access to these browser APIs. Even if malicious scripts somehow execute, they cannot access camera/mic/location without the user being prompted. |
| T-09-09 | Elevation of Privilege | MIME sniffing | mitigate | `X-Content-Type-Options: nosniff` prevents browsers from guessing MIME types. An attacker cannot serve a malicious file disguised as an innocent type (e.g., uploading a .html file that the browser interprets as a script). |
</threat_model>
