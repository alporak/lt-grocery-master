Create `scripts/backup.sh` — a shell script that performs a WAL-safe SQLite backup and prunes old backups.

Implementation per D-01 (local /backups volume), D-02 (.backup API, not cp), D-03 (cron daily), HRDN-05 (daily backup), HRDN-06 (30-day retention):

```bash
#!/bin/sh
set -e

# Configuration
DB_PATH="${DB_PATH:-/app/data/grocery.db}"
BACKUP_DIR="${BACKUP_DIR:-/backups}"
RETENTION_DAYS="${RETENTION_DAYS:-30}"

# Ensure backup directory exists
mkdir -p "$BACKUP_DIR"

# Timestamped backup filename
TIMESTAMP=$(date +%Y%m%d-%H%M)
BACKUP_FILE="$BACKUP_DIR/grocery-$TIMESTAMP.db"

echo "[Backup] Starting backup of $DB_PATH to $BACKUP_FILE"

# Use sqlite3 .backup API — WAL-safe, produces consistent snapshot
# This is SAFE while the database is in use (WAL mode) because it
# uses the SQLite Online Backup API internally. See Pitfall #4.
sqlite3 "$DB_PATH" ".backup '$BACKUP_FILE'"

# Verify backup integrity
echo "[Backup] Verifying backup integrity..."
sqlite3 "$BACKUP_FILE" "PRAGMA integrity_check;" | grep -q "ok"
echo "[Backup] Integrity check passed"

# Record backup size
BACKUP_SIZE=$(du -h "$BACKUP_FILE" | cut -f1)
echo "[Backup] Created: $BACKUP_FILE ($BACKUP_SIZE)"

# Prune backups older than retention period (HRDN-06)
DELETED=$(find "$BACKUP_DIR" -name "grocery-*.db" -mtime +"$RETENTION_DAYS" -delete -print 2>/dev/null | wc -l)
echo "[Backup] Pruned $DELETED backup(s) older than $RETENTION_DAYS days"

# List remaining backups
REMAINING=$(find "$BACKUP_DIR" -name "grocery-*.db" | wc -l)
echo "[Backup] $REMAINING backup(s) retained"
```

Key decisions:
- `DB_PATH`, `BACKUP_DIR`, `RETENTION_DAYS` configurable via env vars (defaults match deployment)
- `PRAGMA integrity_check` after backup — catches corruption immediately (Pitfall #4 detection)
- `find -mtime +30 -delete` for retention — simple, no external tools needed
- `set -e` — script fails on any error (backup failure, integrity failure)
- Uses single quotes around the `.backup` argument to handle paths with spaces
- `du -h` for human-readable backup size — useful in cron logs

**Step 2a — Add backup service to docker-compose.yml:**

Add this new service after the `cloudflared` service (before the file ends):

```yaml
  backup:
    image: alpine:3.21
    entrypoint: ["/bin/sh", "-c"]
    command:
      - |
        apk add --no-cache sqlite
        echo "0 3 * * * /scripts/backup.sh >> /proc/1/fd/1 2>&1" | crontab -
        echo "[Backup] Cron scheduled: daily at 03:00 UTC"
        exec crond -f
    volumes:
      - ./packages/data:/app/data:ro
      - ./backups:/backups
      - ./scripts/backup.sh:/scripts/backup.sh:ro
    restart: unless-stopped
```

Key decisions:
- `alpine:3.21` — minimal image (~7MB), installs only sqlite
- `:ro` on `/app/data` and backup script — read-only for security (backup doesn't need write access to source data)
- `./backups:/backups` — persistent host-mounted directory (survives container restarts, per D-01)
- `${DB_PATH:-/app/data/grocery.db}` in the script uses the mount path directly
- Daily cron at 03:00 UTC (off-peak, minimal write contention)
- `>> /proc/1/fd/1 2>&1` — redirects cron output to container stdout (captured by Docker logging)
- `crond -f` — foreground mode keeps container alive
- `--no-cache` — keeps image small

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Host filesystem → /backups volume | Backup files written to host disk. If host is compromised, backups are accessible. |
| Docker network → backup container | Backup container has no network-exposed ports — reachable only via Docker daemon. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-10-03 | Tampering | /backups volume | accept | Backup files are on host filesystem. Host compromise = backups compromised. Acceptable — full-disk encryption at host level is out of scope. |
| T-10-04 | Denial of Service | backup container (cron) | mitigate | Alpine sidecar uses minimal resources (~10MB RAM). No network listeners. Cron runs once daily. Cannot exhaust host resources. |
| T-10-05 | Elevation of Privilege | backup container | mitigate | Container runs as root (alpine default) but only has read-only access to source data (`:ro` mount). Write access only to dedicated /backups volume. No docker socket mounted. |
</threat_model>
